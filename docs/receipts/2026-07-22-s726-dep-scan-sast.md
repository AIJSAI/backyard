# S-726 dependency-CVE scan + SAST — close receipt

Date: 2026-07-22. Phase 3, step 3 (batched hardening / pre-share safety debt). Adds the two
CI gates a security-first OSS project heading to a public launch needs, and both prove
themselves non-vacuous on every run.

## What it found (the gate earned its keep immediately)

pip-audit, run against the actual synced dependency set, caught **Pillow 11.3.0 carrying a
dozen image-decoder CVEs** (PYSEC-2026-2253…-3454, CVE-2026-54058/59198/59200/59204; fixed
in 12.3.0) — Pillow is the decoder that handles **untrusted uploads and SSRF-fetched
images**, exactly the family-photo attack surface. **Pillow upgraded to 12.3.0**
(`pyproject` pin `>=12.3,<13`, `uv.lock` updated); the media + video + link-preview suites
(90 tests) and the full suite (528) pass on the major bump.

The one remaining advisory, **PYSEC-2026-1845 (pytest)**, is `--ignore-vuln`'d with a
reason: pytest is a dev-only test tool, not in the production runtime, and the pytest-9
major bump would break the pinned `pytest-django` / `pytest-playwright` harness. It is not
shipped to a family box.

## The gates (CI `deps` job)

- **Dependency CVE scan** — `pip-audit` against the installed env (the real synced deps, no
  requirements-file venv). A new advisory on any runtime dependency fails the build.
- **SAST** — `bandit -r src` at medium+ severity/confidence, excluding tests (asserts +
  fixture IPs are B101/B104 noise) and autogenerated migrations. The single production
  medium+ finding — the QR `mark_safe` in `core/handover.py` — is `# nosec`'d at the line
  with its justification (qrcode path geometry over a CSPRNG token, never reflected user
  text; already security-reviewer-cleared in slice 1).
- **Both are non-vacuous (the gitleaks lesson):** a self-test plants a known-vulnerable
  `jinja2==2.11.2` (pip-audit MUST catch it) and a plain `eval(input())` (bandit MUST catch
  it); if either scanner passes its planted issue, the build fails. Verified locally: both
  gates exit 0 clean, both self-tests catch their plant.
