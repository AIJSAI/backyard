## Security review (mandatory for an email wave; isolation worktree)

security-reviewer found **2 HIGH, both fixed before close** â€” the review earned
its keep:

- **HIGH-1 â€” the envelope-trust control was dead in production + tests were
  false-green.** Anymail's Resend inbound handler is the one ESP that never
  populates `AnymailInboundMessage.envelope_recipient` (verified in the installed
  `anymail/webhooks/resend.py`; 10 other ESPs set it), so the original code always
  fell back to the forgeable `To` header, and the tests hand-set a value the
  provider never emits. **Fixed:** the adapter now reads the trusted recipient from
  Resend's own delivery record (`event.esp_event["data"]`, `received_for` else
  `to`); the webhook tests were rewritten to the real event shape so a forged `To`
  genuinely proves the delivered recipient wins. **Verified live below.**
- **HIGH-2 (TS-PP-8) â€” the boot guard did not require `RESEND_INBOUND_SECRET`,**
  leaving a Resend deploy able to boot with an unverifiable (and one refactor from
  unauthenticated) public inbound webhook. **Fixed:** the guard refuses the Resend
  backend without the inbound secret (+test). **Verified live:** the stack boots on
  the Resend backend with the secret; the worker crash-looped *without* it â€” the
  live-repro caught that the secret is deployment-wide, and the compose was fixed to
  pass it to the worker too.
- **LOW-1:** a `message=None` inbound event is dropped rather than raising into a
  Resend retry loop.
